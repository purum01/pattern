Strategy Pattern

"알고리즘 클래스 컬렉션을 정의하고, 각 알고리즘을 개별적으로 캡슐화하여, 이를 서로 교환 가능하게 만드는 것이며, 전략 패턴은 이를 사용하는 클라이언트와는 독립적으로 알고리즘을 변경할 수 있다."

설명
여러 알고리즘(또는 동작 방법)을 인터페이스로 정의하고,
각 알고리즘을 구현 클래스로 캡슐화해서,
클라이언트가 실행 중에도 알고리즘을 자유롭게 교체할 수 있도록 만든다.

🔵 전략 패턴을 사용하는 이유

이유 설명
행동(알고리즘)을 유연하게 교체 실행 중에도 알고리즘(전략)을 자유롭게 바꿀 수 있음
조건문 제거 및 코드 간결화 많은 if-else나 switch문 대신 전략 객체로 분리
OCP(Open-Closed Principle) 준수 기능 확장 시 기존 코드를 수정하지 않고 새로운 전략만 추가
중복 제거 및 응집도 향상 알고리즘을 클래스로 분리해 코드 재사용성과 테스트 용이성 향상

🟠 전략 패턴의 대표적인 활용 사례

활용 분야 구체적 설명
정렬 알고리즘 선택 사용자가 선택한 정렬 기준에 따라 BubbleSort, QuickSort, MergeSort 등을 전략으로 교체
결제 방식 처리 CreditCard, PayPal, KakaoPay 등 다양한 결제 방식 전략을 선택적으로 적용
인증 로직 SNS 로그인, 이메일 로그인, OTP 로그인 등 인증 방식을 전략으로 분리
게임 캐릭터 행동 공격 방식(검, 총, 마법) 등을 전략 객체로 교체 가능
로깅 출력 방식 콘솔 출력, 파일 로그, 원격 로그 전송 등의 전략을 바꿔 적용

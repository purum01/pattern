Singleton Pattern 적용하여 리팩토링하기

📋 실습 목표

주어진 소스코드(리팩토링 전)를 분석하고, 싱글톤 패턴(Singleton Pattern)을 적용하여 인스턴스를 하나만 유지하는 구조로 리팩토링하세요.

생성자를 외부에서 호출하지 못하게 제한하고,
getInstance() 메서드를 통해 항상 같은 객체를 반환하도록 구현하세요.

🛠 리팩토링 지침

  • 생성자를 private으로 선언하여 외부에서 객체 생성을 제한합니다.
  • 클래스 내부에 정적(static) 인스턴스를 선언하여 하나의 객체만 생성되도록 합니다.
  • getInstance() 메서드를 통해 인스턴스를 외부에 제공합니다.
  • 메인에서 여러 번 getInstance()를 호출해도 생성자는 단 한 번만 호출되어야 합니다.

📂 리팩토링 후 파일 구조

/singleton/
      MySingleton.java
      Main.java

🗂 제공 파일 (리팩토링 전)

아래 코드를 복사해 싱글톤 패턴을 적용하세요.

MySingleton.java

package singleton;
  
  public class MySingleton {
      public MySingleton() {
          try {
              Thread.sleep(5000);
          } catch (InterruptedException e) {
          }
          System.out.println("MySingleton()");
      }
  }

Main.java

package singleton;
  
  public class Main {
      public static void main(String[] args) {
          MySingleton my = new MySingleton();
          for (int i = 0; i < 5; i++) {
              my = new MySingleton();
          }
      }
  }

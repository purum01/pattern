Factory Method Pattern 실습

📋 문제 상황

여러분은 다양한 카드를 발급해주는 시스템을 설계하려고 합니다.
- 한쪽에서는 이름이 적힌 ID 카드를 만들어 사용자의 출입을 관리하고 있습니다.
- 다른 쪽에서는 멤버십 카드를 발급해 회원 등급(GOLD, VIP 등)에 따라 혜택을 제공하고자 합니다.
- 하지만 이 두 카드의 생성 방식은 다르고, 발급 후 등록 처리 방식도 각각 다릅니다.
→ 그렇다고 해서 카드마다 새로운 클래스를 직접 생성하는 방식으로 개발을 계속하면 유지보수가 어렵고 코드 중복이 많아질 수 있습니다.

📌 우리가 해결하고자 하는 문제

- 다양한 유형의 카드(IDCard, Membership)를 동일한 방식으로 생성하고 사용할 수 있을까?
- 제품 생성의 책임을 클라이언트가 아닌 팩토리 클래스에 위임할 수 있을까?
- 새로운 카드 유형이 추가되더라도 기존 코드를 수정하지 않고 확장 가능한 구조를 만들 수 있을까?

🛠️ 여러분이 해야 할 일

- 카드의 공통 기능을 추상화한 Product 클래스를 정의합니다.
- 카드 발급 공장을 추상화한 Factory 클래스를 구현합니다.
- 실제로 동작하는 IDCard 와 Membership 클래스를 구현합니다.
- 각각의 카드에 맞는 공장 클래스(IDCardFactory, MembershipFactory)를 정의합니다.
- Main 클래스에서 이 공장들을 통해 카드를 발급하고 사용하는 시나리오를 테스트합니다.

💡 이 실습을 통해 배우는 것

- 팩토리 메서드 패턴의 구조와 원리를 직접 경험합니다.
- 공통된 인터페이스를 사용해 유연하고 확장 가능한 설계를 만드는 방법을 학습합니다.
- 추상 클래스와 다형성을 활용하여 객체 생성과 사용을 분리하는 기법을 익힙니다.

🛠 실습 조건

  • Product.java
    - abstract class 로 선언한다.
    - public abstract void use() 메서드를 선언한다.
    - 이 클래스는 exercise.factorymethod.framework 패키지에 포함되어야 한다.

  • Factory.java
    - abstract class로 선언한다.
    - create(String info) 메서드는 final로 선언하여 재정의할 수 없도록 한다.
    - create(String info) 메서드는 다음 로직을 포함한다:
    1. createProduct(String info) 메서드를 호출하여 제품을 생성한다.
    2. registerProduct(Product product) 메서드를 호출하여 제품을 등록한다.
    3. 생성된 제품을 반환한다.
    - protected abstract Product createProduct(String info) 메서드를 선언한다.
    - protected abstract void registerProduct(Product product) 메서드를 선언한다.
    - 이 클래스는 exercise.factorymethod.framework 패키지에 포함되어야 한다.

  • IDCard.java
    - Product 클래스를 상속받아 구현한다.
    - private String owner 필드를 선언한다.
    - public IDCard(String owner) 생성자를 정의한다.
    1. 매개변수로 받은 이름을 owner 필드에 저장한다.
    2. "IDCard를 만듭니다: [이름]" 을 출력한다.
    - public void use() 메서드를 오버라이딩한다.
    1. "IDCard를 사용합니다: [이름]" 을 출력한다.
    - public String getOwner() 메서드를 정의하여
    1. owner 값을 반환한다.
    - 이 클래스는 exercise.factorymethod.idcard 패키지에 포함되어야 한다.

  • IDCardFactory.java
    - Factory 클래스를 상속받아 구현한다.
    - private List<String> owners 필드를 선언하고, new ArrayList<>()로 초기화한다.
    - protected Product createProduct(String owner) 메서드를 오버라이딩한다.
    1. new IDCard(owner)를 반환한다.
    - protected void registerProduct(Product product) 메서드를 오버라이딩한다.
    1. 전달받은 Product를 IDCard로 캐스팅한다.
    2. 소유자 이름을 owners 리스트에 추가한다.
    - public List<String> getOwners() 메서드를 정의하여, 등록된 소유자 목록을 반환한다.
    - 이 클래스는 exercise.factorymethod.idcard 패키지에 포함되어야 한다.

  • Membership.java
    - Product 클래스를 상속받아 구현한다.
    - private String grade 필드를 선언한다.
    - public Membership(String grade) 생성자를 정의한다.
    1. 매개변수로 받은 등급을 grade 필드에 저장한다.
    2. "Membership 생성됨: 등급 [grade]" 를 출력한다.
    - public void use() 메서드를 오버라이딩한다.
    1. "Membership 사용: 등급 [grade]" 를 출력한다.
    - public String getGrade() 메서드를 정의한다.
    1. grade 값을 반환한다.
    - 이 클래스는 exercise.factorymethod.membership 패키지에 포함되어야 한다.

  • MembershipFactory.java
    - Factory 클래스를 상속받아 구현한다.
    - private List<String> grades 필드를 선언하고, new ArrayList<>()로 초기화한다.
    - protected Product createProduct(String grade) 메서드를 오버라이딩한다.
    1. new Membership(grade)를 반환한다.
    - protected void registerProduct(Product product) 메서드를 오버라이딩한다.
    1. 전달받은 Product를 Membership로 캐스팅한다.
    2. 등급을 grades 리스트에 추가한다.
    - public List<String> getGrades() 메서드를 정의하여, 등록된 회원 등급 목록을 반환한다.
    - 이 클래스는 exercise.factorymethod.membership 패키지에 포함되어야 한다.

🗂 제공 파일

Main.java



public class Main {
    public static void main(String[] args) {
        Factory idFactory = new IDCardFactory();
        Product card1 = idFactory.create("홍길동");
        Product card2 = idFactory.create("이순신");
        card1.use();
        card2.use();

        Factory memberFactory = new MembershipFactory();
        Product m1 = memberFactory.create("GOLD");
        Product m2 = memberFactory.create("VIP");
        m1.use();
        m2.use();
    }
}

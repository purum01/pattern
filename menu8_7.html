팩토리 메서드 패턴(Factory Method Pattern) 실습

📋 실습 목표

객체 생성을 하위 클래스에 위임하여, 상위 클래스는 생성 방식과 무관하게 동작할 수 있도록 팩토리 메서드 패턴을 구현합니다.

상위 클래스 Factory는 템플릿 메서드 역할을 하며,
실제 인스턴스 생성은 하위 클래스 IDCardFactory에서 수행합니다.

🛠 구현 지침

  • Product: 추상 클래스. use() 메서드 정의
  • Factory: 추상 클래스. create(), createProduct(), registerProduct()
  • IDCard: Product를 상속. 이름(owner)을 포함하고 출력
  • IDCardFactory: Factory 상속. 실제 IDCard 인스턴스를 생성 및 등록
  • Main: Factory를 통해 IDCard를 생성 및 사용

🖥 실행 예시


  홍의 카드를 만듭니다.
  강의 카드를 만듭니다.
  박의 카드를 만듭니다.
  홍의 카드를 사용합니다.
  강의 카드를 사용합니다.
  박의 카드를 사용합니다.

📌 UML 다이어그램

🗂 제공 파일

Product.java


package factory;

public abstract class Product {
    public abstract void use();
}

Factory.java


package factory;

public abstract class Factory {
    public final Product create(String owner) {
        Product p = createProduct(owner);
        registerProduct(p);
        return p;
    }

    protected abstract Product createProduct(String owner);
    protected abstract void registerProduct(Product product);
}

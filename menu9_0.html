Composite Pattern

"부분-전체 계층 구조를 표현하기 위해 객체들을 트리 구조로 구성한다. Composite 패턴을 사용하면 클라이언트가 개별 객체와 객체들의 조합을 동일하게 다룰 수 있다."

설명
트리 구조를 만든다. (부분과 전체를 같은 방식으로 다룸)
클라이언트 입장에서는 단일 객체나 복합 객체를 구분하지 않고 사용할 수 있다.

🔵 Composite Pattern 사용 이유

이유 설명
부분-전체 관계를 쉽게 표현하기 위해 트리 구조로 구성된 객체(부품과 전체)를 하나처럼 다룰 수 있게 해준다.
클라이언트 코드를 단순화하기 위해 단일 객체와 복합 객체를 구분하지 않고, 하나의 인터페이스로 처리할 수 있다.
유연하고 확장성 있는 구조를 만들기 위해 새로운 요소 추가나 구조 변경이 쉬워진다. (기존 코드 변경 없이 새 객체 추가 가능)

🟠 Composite Pattern 활용 사례

활용 분야 구체적 설명
파일 시스템 폴더 안에 또 다른 폴더나 파일이 들어갈 수 있다. 폴더와 파일을 동일한 방식으로 다룬다.
GUI 구성요소(컴포넌트) 버튼, 텍스트박스, 패널(다른 컴포넌트를 포함할 수 있음)을 하나의 방식으로 렌더링 처리한다.
문서 편집기 문단, 문장, 단어, 글자를 모두 하나의 컴포넌트처럼 다룬다. (복합 구조)
게임 오브젝트 트리 게임 속 등장인물, 무기, 탈것 등 계층 구조로 관리할 때.

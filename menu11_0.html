Prototype Pattern

"생성할 객체의 종류를 원형 인스턴스로 명시하고, 이 프로토타입을 복사하여 새로운 객체를 생성한다."

설명
객체를 직접 생성하는 대신
존 객체(Prototype)를 복사(Clone) 해서 새로운 객체를 만든다.
비용이 많이 드는 객체 생성을 대신해 복제를 이용해 빠르고 쉽게 객체를 생성할 수 있다.

🔵 Prototype Pattern 사용 이유

이유 설명
객체 생성 비용 절감 복잡하거나 생성 비용이 큰 객체를 매번 새로 생성하는 대신, 기존 객체를 복제해서 빠르게 새 객체를 만들 수 있다.
복잡한 객체 초기화 회피 복잡한 설정이나 초기화 과정을 반복하지 않고, 이미 설정된 프로토타입을 복제하여 동일한 상태를 가진 객체를 얻을 수 있다.
런타임에 객체 생성 컴파일 시점이 아니라 런타임에 필요한 객체를 복제해서 다양한 변형을 만들어낼 수 있다.
클래스에 의존하지 않고 객체 생성 복제는 구체적인 클래스를 알 필요 없이 할 수 있어, 코드의 유연성과 확장성이 높아진다.

🟠 Prototype Pattern 활용 사례

활용 분야 구체적 설명
게임 캐릭터 복제 기본 캐릭터를 하나 만들고, 복제해서 여러 플레이어가 사용하도록 한다. (장비, 능력치만 살짝 수정)
그래픽 편집기 기본 도형(원, 사각형 등)을 복제해서 다양한 위치에 배치할 때 사용한다.
문서 편집기 표준 템플릿 문서(머리말, 꼬리말 등 기본 구성 포함)를 복제해서 새 문서를 만든다.
Spring Framework의 Bean 복제 프로토타입 스코프의 빈(@Scope("prototype"))은 요청할 때마다 새로운 객체를 복제해서 반환한다.
복잡한 데이터베이스 연결 객체 복제 복잡한 설정을 가진 커넥션 객체를 복제해 재사용할 수 있다.

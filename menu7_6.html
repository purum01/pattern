템플릿 메서드 패턴(Template Method Pattern) 구현 실습

📋 실습 목표

출력의 흐름을 일정하게 유지하면서도 세부 동작은 하위 클래스에서 정의할 수 있도록, 템플릿 메서드 패턴을 구현합니다.
공통 알고리즘은 상위 클래스가 제공하고, 세부 구현은 서브클래스에게 위임합니다.

display() 메서드는 상위 클래스에 고정하고,
open(), print(), close()는 하위 클래스에서 구현하세요.

🛠 구현 지침

  • AbstractDisplay: 템플릿 메서드(display())를 구현한 추상 클래스
  • CharDisplay: 문자 하나를 출력하는 하위 클래스
  • StringDisplay: 문자열을 출력하는 하위 클래스
  • 하위 클래스는 open(), print(), close()를 재정의해야 합니다

🖥 실행 예시

<<HHHHH>>
  +----+
  |HAHA|
  |HAHA|
  |HAHA|
  |HAHA|
  |HAHA|
  +----+

📂 파일 구조

/template/
    AbstractDisplay.java   ← 제공됨
    Main.java              ← 제공됨
    CharDisplay.java       ← 직접 구현
    StringDisplay.java     ← 직접 구현

🗂 제공 파일

아래 두 개의 파일을 참고하여 CharDisplay와 StringDisplay를 구현하세요.

AbstractDisplay.java


package template;
  
  public abstract class AbstractDisplay {
      public abstract void open();
      public abstract void print();
      public abstract void close();
  
      public final void display() {
          open();
          for (int i = 0; i < 5; i++) {
              print();
          }
          close();
      }
  }

Main.java


package template;
  
  public class Main {
      public static void main(String[] args) {
          AbstractDisplay d1 = new CharDisplay('H');
          AbstractDisplay d2 = new StringDisplay("HAHA");
          d1.display();
          d2.display();
      }
  }

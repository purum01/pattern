Builder Pattern

"복잡한 객체의 생성 과정을 그 객체의 표현과 분리하여, 동일한 생성 절차가 서로 다른 표현 결과를 만들 수 있도록 한다."

설명
객체 생성 과정과 **객체 구성 방식(표현)**을 분리한다.
동일한 절차를 거치더라도 다른 형태의 객체를 만들어낼 수 있다.
복잡한 객체를 단계별로 조립하는데 유용하다.

🔵 Builder Pattern 사용 이유

이유 설명
복잡한 객체 생성을 단계별로 관리하기 위해 필수 속성과 선택 속성이 섞여 있을 때, 객체 생성 과정을 단계별로 분리해서 명확하게 구성할 수 있다.
생성 과정과 표현을 분리하기 위해 같은 생성 절차를 사용해도 다양한 객체 표현이 가능하게 만들 수 있다.
코드 가독성과 유지보수성을 높이기 위해 생성자 파라미터가 너무 많을 때 (ex: 10개 이상), 빌더를 사용하면 가독성이 좋아지고 코드가 안전해진다.
불변 객체(immutable object) 만들기 위해 빌더를 통해 모든 필드를 설정하고, 최종적으로 한 번에 객체를 생성하여 불변 객체를 쉽게 만들 수 있다.

🟠 Builder Pattern 활용 사례

활용 분야 구체적 설명
복잡한 옵션을 가진 객체 생성 Person 클래스에서 이름, 나이, 주소, 이메일 등 다양한 옵션이 있을 때 빌더 사용.
라이브러리/프레임워크의 설정 객체 Java의 StringBuilder, Spring Framework의 BeanDefinitionBuilder, Lombok의 @Builder
Immutable 객체 생성 생성 후 변경이 불가능한 객체를 만들 때 빌더를 사용하면 안전하고 명확하게 구성할 수 있다.
복잡한 문서(Document) 생성 문서 생성기, 리포트 생성기처럼 다양한 부분이 조립되는 복잡한 데이터 생성 시 유용하다.

Adapter Pattern

"호환성이 없는 인터페이스 때문에 함께 사용할 수 없는 클래스들이 함께 작동하도록 해주는 패턴이다. 어댑터는 클래스의 인터페이스를 클라이언트에서 기대하는 다른 인터페이스로 변환한다."

설명
서로 맞지 않는 인터페이스를 연결해주는 중간 연결자 역할을 하는 것이 어댑터 패턴입이다.

🔵 Adapter Pattern 사용 이유

이유 설명
인터페이스 불일치 해결 기존 클래스의 인터페이스가 새 시스템 요구와 다를 때, 변경 없이 연결할 수 있다.
재사용성 향상 기존 코드를 수정하지 않고 재사용할 수 있다.
코드 수정 최소화 기존 클래스(라이브러리 등)를 건드리지 않고 필요한 방식으로 사용할 수 있다.
새 기능 추가 없이 호환성 확보 클래스에 새로운 메서드를 추가하지 않고도 원하는 방식으로 동작시킬 수 있다.

🟠 Adapter Pattern 활용 사례

활용 분야 구체적 설명
구버전 API와 신버전 시스템 연결 옛날 시스템(구버전 라이브러리)을 새로운 시스템에서 사용할 때
자바 I/O 스트림 InputStream과 Reader 간 연결 (InputStreamReader 클래스가 어댑터 역할)
외부 라이브러리 적용 외부 라이브러리의 구조가 내 애플리케이션 구조와 다를 때 연결
GUI 툴킷 이벤트 처리 서로 다른 이벤트 리스너 인터페이스를 통합할 때

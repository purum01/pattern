Singleton Pattern

"클래스의 인스턴스를 하나만 만들고, 이 인스턴스에 접근할 수 있는 전역적인 접근점을 제공한다."

설명
어떤 클래스가 오직 하나의 인스턴스만 존재하도록 보장하고,
그 유일한 인스턴스를 어디서든 접근할 수 있도록 만든다.

🔵 싱글턴 패턴을 사용하는 이유

이유 설명
하나의 인스턴스만 유지 애플리케이션에서 오직 하나의 객체만 존재해야 할 때 사용
글로벌 접근 지점 제공 어디서든 동일한 인스턴스에 접근 가능 (static 메서드로 제공)
자원 절약 객체 생성을 반복하지 않으므로 메모리와 생성 비용 절감
상태 공유 여러 클래스가 공통된 설정 또는 데이터를 공유해야 할 때 유용

🟠 싱글턴 패턴의 대표적인 활용 사례

활용 분야 구체적 설명
설정 정보 클래스 (Configuration) 앱 전체에서 공통 설정을 읽을 때 (예: DB URL, API 키)
로그 처리기 (Logger) 로그는 중앙에서 일관되게 출력돼야 하므로 하나만 유지
스레드 풀 / 커넥션 풀 하나의 자원 풀에서 여러 클라이언트가 공유
캐시 관리자 자주 사용되는 데이터를 메모리에 올려놓고 여러 곳에서 사용
OS 리소스 관리자 프린터, 그래픽 카드 등 OS 자원은 단일 인스턴스로 관리됨

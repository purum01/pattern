State Pattern

"객체의 내부 상태가 변경될 때, 객체의 행동(Behavior)을 변경할 수 있도록 한다. 객체는 마치 자신의 클래스를 바꾸는 것처럼 보인다."

설명
어떤 객체가 상태에 따라 다른 동작(Behavior) 을 해야 할 때,
상태(State) 객체를 따로 만들어서,
객체가 현재의 상태에 따라 행동을 위임(Delegate) 하게 만든다.

🔵 상태 패턴을 사용하는 이유

이유 설명
조건문 제거 상태에 따라 행동이 달라지는 코드를 if-else나 switch 문 대신 상태 객체로 분리하여 코드가 깔끔해짐
상태 전환의 캡슐화 상태 변경 로직을 상태 객체 내부로 숨겨서 코드 응집도 향상 및 변경 용이성 증가
행위 분리 각 상태에 따른 동작을 별도의 클래스로 나누어, 클래스 단위의 관심사 분리 가능
객체지향 원칙 적용 OCP(개방-폐쇄 원칙), SRP(단일 책임 원칙) 등 SOLID 원칙을 자연스럽게 적용할 수 있음
상태 추가·변경 용이 새로운 상태를 추가하거나 기존 상태를 수정할 때 기존 코드를 거의 건드리지 않고 확장 가능

🟠 상태 패턴의 활용 사례

상황 설명
게임 캐릭터 캐릭터가 '걷기', '뛰기', '앉기', '공격' 상태일 때 각각 다른 행동을 하도록 구현
TCP 연결 TCP 연결은 '연결됨', '연결 중', '끊김' 같은 상태마다 다르게 동작해야 함
문서 편집기 문서가 '읽기 전용', '편집 가능', '잠금' 상태에 따라 다르게 반응하게 함
무기 장전 시스템 총이 '장전 중', '발사 가능', '탄약 없음' 상태에 따라 다른 로직 수행
온라인 주문 시스템 주문이 '결제 완료', '배송 중', '배송 완료', '취소' 상태일 때 행동 변경

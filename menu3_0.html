State Pattern

객체의 내부 상태가 변경될 때, 객체의 행동(Behavior)을 변경할 수 있도록 한다. 객체는 마치 자신의 클래스를 바꾸는 것처럼 보인다.

설명
어떤 객체가 상태에 따라 다른 동작(Behavior) 을 해야 할 때,
상태(State) 객체를 따로 만들어서,
체가 현재의 상태에 따라 행동을 위임(Delegate) 하게 만든다.
이유
복잡한 if-else/switch 제거,
상태별 행동을 독립적으로 관리,
상태 전환을 명확하게 표현
Context(문맥) 객체의 책임 분산
상황 설명
게임 캐릭터 캐릭터가 '걷기', '뛰기', '앉기', '공격' 상태일 때 각각 다른 행동을 하도록 구현
TCP 연결 TCP 연결은 '연결됨', '연결 중', '끊김' 같은 상태마다 다르게 동작해야 합니다.
문서 편집기 문서가 '읽기 전용', '편집 가능', '잠금' 상태에 따라 다르게 반응하게 함
무기 장전 시스템 총이 '장전 중', '발사 가능', '탄약 없음' 상태에 따라 다른 로직 수행
온라인 주문 시스템 주문이 '결제 완료', '배송 중', '배송 완료', '취소' 상태일 때 행동 변경

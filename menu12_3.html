Adapter Pattern 적용하여 리팩토링하기

📋 실습 목표

기존 클래스를 수정하지 않고 새로운 인터페이스에 맞게 사용할 수 있도록 어댑터 패턴을 구현합니다.
상속 방식과 위임 방식의 차이를 비교해보며 적용 능력을 키웁니다.

Banner 클래스는 수정하지 않고,
Print 인터페이스에 맞게 출력 동작을 변환하는 어댑터 클래스를 만들어야 합니다.

🛠 구현 지침

  • Print: 원하는 인터페이스로, printWeak(), printStrong() 메서드 선언
  • Banner: 기존 클래스. showWithParen(), showWithAster() 메서드를 가짐
  • PrintBanner: Banner를 상속하고 Print 인터페이스를 구현 (상속 기반 어댑터)
  • AnotherPrintBanner: Banner를 내부에 포함하고 Print 구현 (위임 기반 어댑터)
  • Main에서는 Print 타입만을 사용하여 출력

🖥 실행 예시

(Hello)
  *Hello*

📂 파일 구조

/adapter/
    Print.java                ← 제공됨
    Banner.java               ← 제공됨
    PrintBanner.java          ← 구현 대상 (상속 방식)
    AnotherPrintBanner.java   ← 구현 대상 (위임 방식)
    Main.java                 ← 제공됨

📌 UML 다이어그램

🗂 제공 파일

Print.java

package adapter;
  
  public interface Print {
      public void printWeak();
      public void printStrong();
  }

Banner.java

package adapter;
  
  public class Banner {
      String string;
  
      public Banner(String string) {
          this.string = string;
      }
  
      public void showWithParen() {
          System.out.println("(" + string + ")");
      }
  
      public void showWithAster() {
          System.out.println("*" + string + "*");
      }
  }

Main.java

package adapter;
  
  public class Main {
      public static void main(String[] args) {
          Print p = new PrintBanner("Hello");
          p.printWeak();
          p.printStrong();
      }
  }

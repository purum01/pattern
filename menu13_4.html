✅ 프록시 패턴 활용 방법

  1. 비즈니스와 무관한 기능 분리
    인증, 모니터링, 트래픽 제어 등 공통 부가 기능을 비즈니스 로직과 분리하여 개발 가능.
    Spring AOP와 결합 시 생산성과 유지보수성 향상.
  2. RPC(원격 호출)에서의 활용
    클라이언트가 직접 호출하지 않고 RPC 프록시 객체를 통해 간접 호출.
    원격 호출 방식과 무관하게 로컬 객체처럼 사용 가능.
  3. 캐시 기능에 적용
    동일 요청에 대해 반복 계산하지 않고 캐시된 결과를 반환.
    예: ?cached=true 같은 조건에 따라 Spring AOP로 캐시 처리 가능.

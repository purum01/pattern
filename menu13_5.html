Proxy Pattern 적용하여 리팩토링하기

📋 실습 목표

인스턴스의 생성을 지연시키거나 접근 제어를 하기 위해 프록시 패턴을 구현합니다.
실제 객체와 동일한 인터페이스를 사용하며, Printer 객체의 생성을 PrinterProxy를 통해 지연시킵니다.

Printable 인터페이스를 통해 Printer와 PrinterProxy가 동일한 기능을 가지며,
프록시 객체는 실제 객체 생성을 나중에 수행합니다 (지연 초기화).

🛠 구현 지침

  • Printable: 인터페이스. setPrinterName(), getPrinterName(), print() 포함
  • Printer: 실제 무거운 객체로, 생성 시 시간이 오래 걸림 (예: Thread.sleep)
  • PrinterProxy: 이름만 저장하고 실제 객체는 필요할 때 생성
  • Main: Printable 타입을 통해 프록시를 사용

🖥 실행 예시

이름은 현재 Alice입니다.
  이름은 현재 Bob입니다.
  Printer의 인스턴스를 생성중.....완료
  === Bob ===
  Hello, world.

📂 파일 구조

/proxy/
    Printable.java        ← 제공됨
    Printer.java          ← 구현 대상 (실제 대상)
    PrinterProxy.java     ← 구현 대상 (프록시)
    Main.java             ← 제공됨 (사용 예시)

📌 UML 다이어그램

🗂 제공 파일

Printable.java

package proxy;
  
  public interface Printable {
      public void setPrinterName(String name);
      public String getPrinterName();
      public void print(String string);
  }

Main.java

package proxy;
  
  public class Main {
      public static void main(String[] args) {
          Printable p = new PrinterProxy("Alice", "proxy.Printer");
          System.out.println("이름은 현재" + p.getPrinterName() + "입니다.");
          p.setPrinterName("Bob");
          System.out.println("이름은 현재" + p.getPrinterName() + "입니다.");
          p.print("Hello, world.");
      }
  }

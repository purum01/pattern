🧪 상태 패턴 실습 안내

🎯 실습 목표

  • 상태(State)에 따라 동작이 달라지는 객체를 설계하고 구현합니다.
  • 조건문 대신 상태 객체에 행동을 위임합니다.
  • OCP(개방-폐쇄 원칙)를 고려한 유연한 설계를 경험합니다.

📁 제공 파일 목록

  • SafeFrame.java - 버튼 기반 GUI
  • Main.java - 프로그램 진입점 (시각 시뮬레이션)
  • Context.java - 상태 전환/로그 인터페이스
  • State.java - 상태 객체 인터페이스
  • DayState.java - 낮 상태 구현
  • NightState.java - 밤 상태 구현

🛠️ 실습 조건

  1. 모든 상태는 State 인터페이스를 구현해야 합니다.
  2. 상태에 따라 버튼 행동이 다르게 동작해야 합니다.
  3. 시간이 09~17시가 아니면 야간 상태로 전환합니다.
상태 시간 변화 금고 사용 비상벨 일반 통화
주간 09~17시 유지, 이외는 야간 전환 기록 로그 보안센터 호출 보안센터 호출
야간 09~17시 진입 시 주간 전환 비상 로그 기록 보안센터 호출 통화 녹음

✅ 결과 확인 방법

  • 콘솔 로그 및 텍스트 화면에서 상태 전이와 반응을 확인합니다.
  • 예시: [야간]에서[주간]로 상태가 변화했습니다.

💡 추가 도전 과제

  • 저녁 상태 또는 휴일 상태 추가
  • 버튼 동작을 커스터마이징 해보기
  • GUI 디자인 개선
